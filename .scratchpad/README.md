# Persistent Working Memory

`.scratchpad/` は、AIエージェントがセッションをまたいで参照する永続的なWorking Memoryです。Git管理対象ですが、Git履歴の代わりではありません。コードから明らかな情報、ファイル単位の変更履歴、コミット内容の複製は記録せず、コードやGit履歴だけでは分からない文脈を残します。

## ファイル一覧

| ファイル | 用途 | 更新タイミング |
| --- | --- | --- |
| `current.md` | 現在進行中の作業、未解決事項、次に行う処理 | 各作業セッション |
| `backlog.md` | 将来の案、延期された作業、未着手の改善 | 案が出たとき |
| `completed.md` | 完了した作業の短い記録 | 作業完了時 |
| `decisions.md` | コードだけでは分からない設計判断と理由 | 判断を行ったとき |
| `lessons.md` | 失敗した方法、再発防止、再利用可能な知見 | 学びが発生したとき |
| `quirks.md` | 実行環境固有の不具合、タイミング問題、回避策 | 発見または修正時 |

## current.mdのルール

`current.md`は小さく保ち、目安として40行以内にします。現在作業中の内容だけを記載し、完了した作業の経緯や長いログは残しません。作業完了時は必要な情報を他のファイルへ移して、完了した内容を削除します。

推奨形式：

```markdown
# Current Work

## Task: タスク名

- Goal:
- Status:
- Files involved:
- Open questions:
- Next action:
```

不要な項目は省略できます。

## backlog.mdのルール

まだ着手していない内容だけを記載します。作業を開始した項目は `current.md` へ移して `backlog.md` から削除し、実装しないと決定した項目も削除します。同じ内容を `current.md` と `backlog.md` の両方に残しません。

## completed.mdのルール

新しい項目を上に追加する逆時系列とします。一件につき一文または二文とし、細かい変更ファイル一覧やコミットハッシュは記載しません。日付形式はプロジェクト内で `YYYY-MM-DD` に統一します。

推奨形式：

```markdown
# Completed

- 2026-07-23: Added persistent agent working memory under `.scratchpad/` and documented project-wide AI agent rules.
```

## decisions.mdのルール

実装内容ではなく、なぜその実装を選んだかを残します。単純な命名変更やコードから自明な判断は記録しません。

## lessons.mdのルール

一度失敗した方法や、今後の作業にも適用できる知見を記載します。一時的なエラー全文や大量のログは貼り付けません。

## quirks.mdのルール

OS、ランタイム、外部サービス、ライブラリ、タイミングなどに依存する挙動を記載します。可能な範囲で発生条件、症状、原因、回避策、解消済みかどうかを含めます。

## 共通ルール

- コミットハッシュを記載しない。
- ファイル単位の変更履歴を作らない。
- Git履歴の内容を複製しない。
- 長い会話ログを保存しない。
- 同じ内容を複数ファイルへ重複して書かない。
- 存在しないファイルや削除済みコードを参照する記録は削除する。
- 古くなった回避策や既に不要な制約は削除する。
- 秘密情報を保存しない。
- 作業者個人の感想ではなく、次の作業者に必要な情報を書く。
- 推測と確認済み事項を区別する。
